ocgt-monitor

OpenCode Go 套餐额度监控工具 — 一步一步教你用

这个工具是做什么的？
你平时用 Claude Code 的时候，背后是通过 ocgt 代理连接 OpenCode Go 的服务。 ocgt-monitor 大部分功能与 ocgt 无关，它直接调用 OpenCode Go 的接口查询额度。 只有 history（查看历史消耗）需要读取 ocgt 产生的日志文件。

先搞懂几个词

终端 / 命令行 / cmd — 就是那个黑底白字的窗口。Windows 里叫"命令提示符"。按 Win 键，输入 cmd 回车就能打开。我们的 ocgt-monitor.exe 是命令行程序，所以需要在终端里输入命令来运行它。

为什么有 exe 还要打开终端？ 因为 ocgt-monitor 有 11 种不同功能（查额度、查余额、看历史、启动网页…），双击 exe 只能执行"帮助"。你需要打开终端，输入不同的"命令"来告诉它你想查哪一个。

Cookie — 你在网站登录后，浏览器存的一个"通行证"。我们需要把它复制出来，ocgt-monitor 才能代替你去查额度。

准备工作

备注： quota（查额度）和 balance（查余额）完全不需要 ocgt，直接调用 OpenCode Go 接口。
只有 history（看历史消耗）需要 ocgt 产生的日志文件（在 ~/.ocgt/logs/ 目录）。

第一步：打开终端

Step 1

按键盘上的 Win 键 → 输入 cmd → 按回车。

Microsoft Windows [版本 10.0.xxxxx]
(c) Microsoft Corporation。保留所有权利。
C:\Users\你的用户名>_

那个 C:\Users\你的用户名> 就是"提示符"，等着你输入命令。

第二步：进入程序所在文件夹

Step 2

在终端里输入下面这行命令，然后按回车：

C:\Users\你的用户名> cd C:\Users\14050\Desktop\opencodego_tokens\ocgt-monitor

输完回车后，提示符会变成：

C:\Users\14050\Desktop\opencodego_tokens\ocgt-monitor>

出现这个变化说明你已经进入这个文件夹了。之后所有命令都在这个提示符后面输入。

如果你把 ocgt-monitor.exe 放在了别的位置，这里要改成你实际的路径。

第三步：获取 Cookie 和 Workspace ID

Step 3
  1. 打开 Chrome 浏览器，登录 opencode.ai
  2. 按下键盘上的 F12 键，右侧会弹出一个面板
  3. 点击顶部 Application（应用）选项卡
  4. 左侧找到 Cookies → 点击 https://opencode.ai
┌─ F12 面板 ──────────────────────────
│ Elements │ Console │ Application │ Network │
│ ────────────────────────────────────
│ Storage: │ Name │ Value
│ Local Storage │ session │ xxx...
│ ▶ Cookies │ ... │ ...
│ https://opencode.ai ← 点这里
└─────────────────────────────────────
  1. 在 Name 列表里随便点一个，按 Ctrl+A 全选整个表格，再按 Ctrl+C 复制
  2. 粘贴到记事本里存着（Cookie 字符串很长，不用担心）

获取 Workspace ID： 在浏览器地址栏找到 /workspace/wrk_xxxxxxxxxxxx/usage 这段，把 wrk_ 开头的那串复制下来。

第四步：运行配置向导

Step 4

在终端里输入：

C:\Users\...\ocgt-monitor> ocgt-monitor config init

按回车后，它会依次问你问题。你按下面这样回答：

1. 看到"按回车继续" → 直接按一下回车

2. 问 Cookie → 在终端里点右键（等于粘贴），把刚才复制的 Cookie 贴进去，按回车

3. 问 Workspace ID → 输入 wrk_xxxxxxxxxxxx，按回车

4. 问 DeepSeek API Key → 直接按回车跳过（除非你有 DeepSeek 账户）

看到 "配置已保存！" 就说明成功了。

配置只需要做一次。以后直接查额度就行，不用重新配置。

第五步：查套餐额度

Step 5

在终端里输入：

C:\Users\...\ocgt-monitor> ocgt-monitor quota

如果一切正常，你会看到：

========================================
  OpenCode Go 套餐额度
----------------------------------------
  Rolling: [████████████░░░░░░] 65%  reset in 2h
  Weekly:  [████████████████░░] 82%  reset in 3d
  Monthly: 无限额度
========================================

怎么看： Rolling=滚动额度(短期)，Weekly=本周额度，Monthly=本月额度。进度条越长说明用得越多。

如果看到红色错误提示，说明 Cookie 可能过期了。回到第三步重新复制，再运行 config init 更新。

更多常用命令

ocgt-monitor balance查 DeepSeek 余额（如果设置了 API Key）
ocgt-monitor history看最近 7 天每天用了多少 Token
ocgt-monitor watch持续监控，每 60 秒自动刷新。按 Ctrl+C 退出。
ocgt-monitor serve启动网页版仪表盘。然后浏览器打开 http://127.0.0.1:8788

每次都要先 cd 进文件夹吗？

是的。每次打开新终端，第一步都是：

C:\Users\你的用户名> cd C:\Users\14050\Desktop\opencodego_tokens\ocgt-monitor

然后才能运行 ocgt-monitor quota 等命令。

发给我朋友用，怎么清空我的配置？

ocgt-monitor config delete default
运行这个命令后，你的 Cookie 等所有信息都会被清除。然后就可以放心把 ocgt-monitor.exe 发给你朋友了。

遇到问题？

Q: 输入 ocgt-monitor 提示"不是内部或外部命令"
你没有先进入程序所在文件夹。先运行 cd C:\Users\14050\Desktop\opencodego_tokens\ocgt-monitor。
Q: Cookie 粘贴后没反应？
在终端里点右键就是粘贴。如果还不行，在终端标题栏点右键 → 编辑 → 粘贴。
Q: 配置好了但查 quota 还是报错？
Cookie 可能过期了。重新打开浏览器登录 opencode.ai，复制 Cookie，再运行 config init 更新。
Q: history 显示"暂无数据"？
ocgt 代理还没有产生日志。先正常使用 Claude Code 对话，过一会儿再查就有了。
Q: 我只有一个 exe，怎么会有网页界面？
运行 ocgt-monitor serve 后，浏览器输入 http://127.0.0.1:8788 就能访问网页界面。网页是内嵌在 exe 里面的，不需要额外下载东西。
Q: 这个 Cookie 会过期吗？
会。通常几天到几周不等。过期后重新登录 opencode.ai 复制新的就行。
